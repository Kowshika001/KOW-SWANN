## Task 1: Detection of Geometric Shapes in a Grayscale Image

### 1.1 Approach Summary

The image is first converted into a grayscale representation. To detect individual objects, a Breadth-First Search (BFS) based traversal is used to identify connected components of pixels with intensity above a certain threshold.

Once a component is detected, it is analyzed for shape classification.

### 1.2 Rectangle Detection

For each detected region, a bounding box is computed. If the area of the bounding box is exactly equal to the total number of pixels in the region (`width * height == pixel_count`), the region perfectly fills the box and is classified as a rectangle.

### 1.3 Circle Detection

Circular shapes are identified using circularity. First, the bounding box is checked to ensure it's not too elongated. Then, the circularity is computed using the formula:
`Circularity = (4 * PI * Area) / (Perimeter^2)`

A value close to 1.0 indicates a perfect circle. Regions with circularity above a threshold (e.g., 0.7) are classified as circular.

### 1.4 Filtering and Output

Regions classified as either a rectangle or a circle are considered "valid shapes." The final output image is generated by preserving the pixels of these valid shapes and suppressing all others.

### 1.5 Complexity Analysis

- **Time Complexity**: `O(M × N)`, as the BFS traversal visits each pixel at most once.
- **Space Complexity**: `O(M × N)`, for the `visited` matrix and BFS queue.

## Task 2: Sparse Matrix Representation and Image Processing

### 2.1 Sparse Matrix Representation

The binary image is converted into a sparse representation using a singly linked list (`std::list`). Each node stores the `row`, `column`, and `value` of a foreground pixel (value '1'). This significantly reduces storage for images with few foreground pixels.

### 2.2 Detection of Connected Components

A Breadth-First Search (BFS) traversal identifies distinct objects. Starting from an unvisited foreground pixel, the BFS explores all connected neighbors (up, down, left, right). To make this efficient on a sparse structure, a `std::set` of all pixel coordinates is used for quick neighbor lookups.

### 2.3 Area and Boundary Calculation

- **Area**: For each detected component (region), the area is the total number of pixels belonging to it, which is naturally found during the BFS traversal.
- **Boundary**: A pixel is a boundary pixel if at least one of its four immediate neighbors does not belong to the same region.

### 2.4 Image Reconstruction

The sparse representation is converted back into a matrix. To visualize the detected components, the reconstructed matrix is filled with region-specific IDs (e.g., '1' for all pixels in region 1, '2' for region 2, etc.).

### 2.5 Complexity Analysis

- **Time Complexity**: `O(M × N)`, dominated by the initial matrix scan to build the sparse list.
- **Space Complexity**: `O(M × N)`, primarily for the `visited` map used during BFS.
