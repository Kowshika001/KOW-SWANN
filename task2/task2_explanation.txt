## Task 2 Report: Sparse Matrix Image Processing

### 1. Approach Summary

This task required representing a binary image using a sparse matrix format (specifically a linked list) and performing several image processing operations on this representation. The primary challenge is that a simple linked list is not inherently structured for efficient 2D spatial queries (like finding neighbors). My approach uses the linked list as the primary storage but employs helper data structures (like maps and sets) within methods to perform the required operations efficiently.

Here's a breakdown of the implementation:

1.  **Sparse Representation**:
    *   A `struct SparseNode { int row, col; }` stores the coordinates of a non-zero (i.e., '1') pixel.
    *   The `SparseImage` class holds the image dimensions and a `std::list<SparseNode>` which acts as our primary linked list structure.
    *   A constructor converts a standard `std::vector<std::vector<int>>` (dense matrix) into this sparse list by iterating through the dense matrix and adding a `SparseNode` for each '1' found.

2.  **Object Identification (Connected Components)**:
    *   To find distinct objects, I implemented a **Union-Find** (or Disjoint Set Union) algorithm. This is highly efficient for this type of problem.
    *   To adapt it to the sparse structure, I first create a `std::map` that allows quick lookups of pixels by a unique key (`row * cols + col`). This avoids O(N) traversals of the linked list just to find a pixel's neighbor.
    *   The algorithm iterates through each node in the linked list. For each pixel, it checks its top and left neighbors. If a neighbor exists (is a '1' pixel), it performs a `union` operation on the two pixels' sets.
    *   After iterating through all pixels, the Union-Find structure correctly partitions all pixels into disjoint sets, where each set represents a distinct object.
    *   The area of each object is then calculated by counting the number of pixels belonging to each final set.

3.  **Boundary Pixel Detection**:
    *   A pixel is on the boundary if at least one of its four direct neighbors is a background ('0') pixel.
    *   To implement this efficiently, I first populate a `std::set` with the unique keys of all '1' pixels. A `set` provides fast O(log P) lookup, where P is the number of '1' pixels.
    *   I then iterate through each node in my sparse linked list. For each node `(r, c)`, I check the four neighbor coordinates. If a neighbor's key is *not* found in the `set`, it must be a background pixel. Therefore, `(r, c)` is a boundary pixel.

4.  **Pixel Flipping**:
    *   Flipping the image (1s to 0s and 0s to 1s) on a sparse structure means creating a new set of nodes representing the inverted image.
    *   My implementation creates a temporary dense matrix initialized entirely with '1's. It then iterates through the *original* sparse list and sets the corresponding pixels in this temporary matrix to '0'.
    *   Finally, a new `SparseImage` object is created from this temporary flipped matrix. This approach correctly and efficiently generates the new sparse list for the flipped image.

5.  **Image Reconstruction**:
    *   This is the reverse of the initial conversion. A `toDenseMatrix()` method creates an M x N matrix initialized to all '0's. It then iterates through the sparse linked list and places a '1' at the `(row, col)` coordinates of each node.

### 2. Time and Space Complexity

Let M, N be the image dimensions, and P be the number of non-zero pixels ('1's).

*   **Conversion to Sparse (Initial)**
    *   Time: O(M * N) - Must scan the entire input matrix once.
    *   Space: O(P) - To store the P non-zero pixels in the linked list.

*   **Object Identification (Union-Find)**
    *   Time: O(P * α(P)), where α is the extremely slow-growing inverse Ackermann function. With the `std::map` for lookups, each find/union is nearly constant time. This is highly efficient.
    *   Space: O(P) - For the Union-Find map and the helper lookup map.

*   **Boundary Detection**
    *   Time: O(P * log P) - Dominated by inserting P items into a `std::set` and then P lookups.
    *   Space: O(P) - To store the keys in the `std::set`.

*   **Reconstruction to Dense**
    *   Time: O(M*N) - To initialize the dense matrix.
    *   Space: O(M*N) - For the output dense matrix.
