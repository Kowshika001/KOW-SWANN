## Task 1 Report: Object Detection in a Grayscale Image

### 1. Approach Summary

The core of the problem is to identify connected regions of pixels that share a common characteristic—in this case, having a pixel intensity above a certain threshold. This is a classic "connected components" or "blob detection" problem. My approach uses a Breadth-First Search (BFS) algorithm, a fundamental graph traversal technique, to find these regions.

The algorithm works as follows:

1.  **Initialization**: A 2D boolean matrix `visited` of the same dimensions as the input image is created to keep track of pixels that have already been assigned to an object. An intensity `threshold` is defined to distinguish object pixels from the background.

2.  **Matrix Traversal**: The program iterates through every pixel of the image from top-left to bottom-right.

3.  **Object Discovery**: If a pixel `(r, c)` is found that has an intensity greater than the `threshold` and has not been `visited`, it is considered the starting point of a new potential object.

4.  **BFS for Component Finding**: A BFS is initiated from this starting pixel:
    *   A queue is used to store pixels that need to be checked. The starting pixel is added to the queue and marked as `visited`.
    *   While the queue is not empty, a pixel is dequeued. This pixel is added to the current object's list of pixels. Its coordinates are used to update the object's bounding box (min/max row and column).
    *   Its four neighbors (up, down, left, right) are examined. If a neighbor is within the image bounds, has an intensity above the `threshold`, and has not been `visited`, it is marked as `visited` and enqueued.
    *   This process continues until the queue is empty, at which point all connected pixels of similar intensity have been found and grouped into one object.

5.  **Object Classification and Output**:
    *   Once a complete object is found, its properties are analyzed. I calculate the `density` of the object within its bounding box (i.e., `actual pixel count / bounding box area`).
    *   A high density (e.g., > 0.9) strongly suggests a solid, rectangular shape, as most of the space in the bounding box is filled.
    *   A lower density can indicate an irregular or circular shape. For a perfect circle within a square bounding box, the density would be approximately π/4 ≈ 0.785. This heuristic is used for a tentative classification.
    *   Finally, the program generates a character-based visual output, drawing a `+`, `|`, and `-` bounding box around the original pixel locations (`#`).

### 2. Time and Space Complexity

*   **Time Complexity**: **O(M * N)**, where M is the number of rows and N is the number of columns in the image. This is because the main loop iterates through each pixel once. The BFS traversal for each object also visits each pixel within that object's component at most once. Since each pixel can only belong to one object, every pixel is processed a small, constant number of times in total.

*   **Space Complexity**: **O(M * N)**. This is dominated by the `visited` matrix, which requires space proportional to the image size. In the worst-case scenario (an image that is one single, large object), the queue used by BFS could also potentially hold all M*N pixels. The storage for the detected objects' pixels is also, at most, O(M*N) in total.
